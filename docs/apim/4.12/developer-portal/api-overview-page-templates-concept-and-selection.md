# API Overview Page Templates: Concept and Selection

## Overview

API Overview Page Templates provide pre-configured Markdown content for API documentation pages in the Gravitee Developer Portal. Two templates are available: a standard template for traditional APIs and an MCP-specific template for Model Context Protocol proxy APIs. When an API is published to the Developer Portal, the platform automatically selects the appropriate template based on API type. Both templates populate with API metadata and use custom web components for visual consistency.

## Key Concepts

### Standard API Template

The standard template introduces the API through the Developer Portal lens, emphasizing subscription-based access and interactive documentation. It displays API metadata (version, visibility, owner, deployment date) in a styled card and organizes guidance into three areas: subscribing to plans, exploring documentation, and integrating with client applications. The template uses a three-column grid layout with feature cards and includes customization suggestions for API publishers.


### MCP Proxy API Template

The MCP template is tailored for Model Context Protocol servers published through the Gravitee gateway. It explains how to connect AI assistants (Cursor, VS Code, Claude Desktop) to the MCP server using a one-click install component. The template highlights secure access through gateway policies and presents three focus areas: discovering tools and resources, understanding gateway-enforced security, and subscribing for credentials. It includes a link to MCP-specific authentication documentation.
