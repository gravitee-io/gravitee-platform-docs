### Related Changes

The REST API exposes five new endpoints under `/applications/{application}/certificates` with standard CRUD operations. Database schema includes a new `client_certificates` table (JDBC) or collection (MongoDB) with indexes on `application_id`, `environment_id`, `fingerprint`, and `status`. The repository layer implements `existsByFingerprintAndActiveApplication` to enforce uniqueness constraints and `deleteByApplicationId` for cascade deletion. Error handling introduces two new exceptions: `ClientCertificateNotFoundException` (404, technical code `clientCertificate.notFound`) and `ClientCertificateAlreadyUsedException` (400, technical code `clientCertificate.alreadyUsed`). A pagination offset bug was fixed in PR #15283 to correctly convert 1-based API page numbers to 0-based repository queries.
