
# Enforcing Logging Standards with ArchUnit

The `gravitee-archrules-maven-plugin` validates logging conventions at build time using ArchUnit rules.
 This plugin ensures that custom policies and plugins adhere to node-aware logging standards.

### Prerequisites

Before enabling ArchUnit validation, ensure your project meets the following requirements:

* `gravitee-parent` version 24.0.0 or later
* `gravitee-archrules-maven-plugin` version 1.0.0-alpha.2 or later

### Enable ArchUnit validation

1. Add the plugin to your `pom.xml`. The plugin is inherited from `gravitee-parent` 24.0.0+.
2. Run `mvn verify` to execute ArchUnit checks during the Maven verify phase.
3. Configure exemptions for legacy classes if needed:
   * Use `LoggingArchitectureRules.configure().allowIn(Set.of("LegacyClass"))` to exempt specific classes.
   * Use `excludePackagesFromScan("io.gravitee.legacy..")` to exclude entire packages.
4. To skip checks in CI pipelines, pass `-Dgravitee.archrules.skip=true` to Maven.

### ArchUnit rules

The plugin enforces the following logging rules:

* **No direct SLF4J LoggerFactory usage**: All classes must use `NodeLoggerFactory.getLogger()` instead of `org.slf4j.LoggerFactory.getLogger()`.
* **Context-aware logging in reactive handlers**: Reactive handlers with `ExecutionContext` access must use `ctx.withLogger(log)` before logging. Direct logger calls will fail validation.

### Automatic exclusions

The following classes and packages are automatically excluded from ArchUnit validation:

* Classes ending with `ConfigurationEvaluator` (generated by `gravitee-plugin-annotation-processors`)
* Classes in `io.gravitee.gateway.reactive.api..` packages
* Classes in `io.gravitee.gateway.api..` packages
