# Node Logging Migration Guide and Version History

## Related Changes

The `gravitee-archrules-maven-plugin` was introduced in version 1.0.0-alpha.1 and updated to 1.0.0-alpha.2, with two executions (`global-logging-check` and `execution-context-logging-check`) running during the Maven `verify` phase. CI pipelines add `-Dgravitee.archrules.skip=true` to Maven commands during Nexus staging, snapshot publishing, release builds, and integration tests. The default value of `node.logging.pattern.overrideLogbackXml` changed from `false` to `true` in version 8.0.0-alpha.15. Default log patterns now include milliseconds (`HH:mm:ss.SSS` instead of `HH:mm:ss`) as of version 8.0.0-alpha.14. The `MdcListConverter` registration was migrated from the deprecated `DEFAULT_CONVERTER_MAP` to `PatternLayout.DEFAULT_CONVERTER_SUPPLIER_MAP` in version 8.0.0-alpha.14 to avoid classloader issues. Class hierarchy caching was added to `AbstractBaseExecutionContextAwareLogger` in version 5.0.0 to optimize log source registration via reflection. The `gravitee-reactor-native-kafka` module removed static context class registration in version 6.0.0-alpha.7, relying on dynamic log source discovery instead.
