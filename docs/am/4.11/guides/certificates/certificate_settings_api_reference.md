### Related Changes

The Management API now includes a dedicated endpoint for updating certificate settings (`PUT /certificate-settings`) that avoids full domain reloads. The event system emits `DOMAIN_CERTIFICATE_SETTINGS.UPDATE` events to notify gateway nodes of certificate settings changes without requiring domain synchronization. Certificate deletion validation now includes a check for fallback certificate usage, returning HTTP 400 with `CertificateIsFallbackException` when attempting to delete a fallback certificate. The UI certificate selection component now displays system certificates in the fallback certificate dropdown. Database schema migration `4.11.0-update-domain-certificate-settings` adds a `certificate_settings` CLOB column to the `domains` table. Warning-level logs are emitted when fallback certificates are used, including the original certificate ID and the fallback certificate ID being substituted.
